# 전체 진행 설명서

작성일: 2026-06-20

## 1. 문서 목적

이 문서는 `img` 폴더에 들어 있는 이미지 파일(`*.jpg`)을 분석해서 Python 실습 코드까지 완성한 전체 과정을 타임라인 형태로 설명한다.

개발자가 아닌 Python 초입 입문자도 이 문서를 보고 순서대로 따라 하면 다음 결과물을 만들 수 있도록 작성했다.

- 이미지 분석 보고서: `01_report.md`
- 해야 할 일 정리 문서: `02_task.md`
- Python 실습 코드: `src/practice.py`
- 완료 검증 보고서: `03_compelete.md`
- 전체 진행 설명서: `04_description.md`

이 문서는 단순한 결과 요약이 아니라, 무엇을 왜 해야 하는지, 어떻게 해야 하는지, 어떤 명령어로 확인해야 하는지까지 설명한다.

## 2. 최종 폴더 구조

최종적으로 프로젝트 폴더는 다음과 같은 형태가 된다.

```text
.
├── 01_report.md
├── 02_task.md
├── 03_compelete.md
├── 04_description.md
├── img
│   ├── s1.jpg
│   ├── s2.jpg
│   ├── s3.jpg
│   ├── s4.jpg
│   └── s5.jpg
└── src
    └── practice.py
```

각 파일의 의미는 다음과 같다.

| 파일 | 역할 |
| --- | --- |
| `img/s1.jpg` ~ `img/s5.jpg` | 분석 대상 이미지 |
| `01_report.md` | 이미지 내용을 상세히 판독한 보고서 |
| `02_task.md` | 이미지와 보고서를 바탕으로 실제로 해야 할 일을 정리한 문서 |
| `src/practice.py` | 해야 할 일을 Python 코드로 구현한 실습 파일 |
| `03_compelete.md` | 코드 생성과 검증이 완료되었음을 기록한 완료 보고서 |
| `04_description.md` | 전체 진행 과정과 따라 하기 방법을 설명하는 문서 |

## 3. 전체 진행 타임라인

### 1단계: 이미지 파일 확인

먼저 `img` 폴더 안에 어떤 이미지가 있는지 확인했다.

확인 대상은 다음 5개 파일이다.

```text
img/s1.jpg
img/s2.jpg
img/s3.jpg
img/s4.jpg
img/s5.jpg
```

왜 이 작업을 하는가?

이미지 분석을 시작하기 전에 실제 분석 대상 파일이 존재하는지 확인해야 한다. 파일이 없거나 이름이 다르면 이후 보고서와 코드 작성 기준이 틀어질 수 있다.

초보자가 직접 확인하려면 터미널에서 다음 명령어를 입력한다.

```bash
find img -maxdepth 1 -type f
```

또는 간단히 다음 명령어도 사용할 수 있다.

```bash
ls img
```

정상이라면 `s1.jpg`부터 `s5.jpg`까지 보인다.

### 2단계: 이미지 내용 분석

각 이미지를 열어 화면에 적힌 내용을 읽고 분석했다.

분석 결과 이미지는 크게 두 개의 과제를 담고 있었다.

1. 작업 리스크 및 우선순위 분석 프로그램
2. 통합 운영 대시보드 시스템

`img/s1.jpg`, `img/s2.jpg`는 첫 번째 과제인 작업 리스크 분석 과제였다.

`img/s3.jpg`, `img/s4.jpg`, `img/s5.jpg`는 두 번째 과제인 통합 운영 대시보드 과제였다.

왜 이 작업을 하는가?

이미지는 단순 참고 자료가 아니라 실제 실습 코드의 요구사항이다. 따라서 이미지 안에 있는 입력 데이터, 계산 기준, 출력 예시, 목표치를 정확히 읽어야 한다.

이 단계에서 확인해야 하는 항목은 다음과 같다.

- 어떤 데이터를 입력으로 사용하는가
- 어떤 계산을 해야 하는가
- 어떤 기준으로 정상, 주의, 위험을 판단하는가
- 결과를 어떤 형식으로 출력해야 하는가
- 이미지 안에 모순되거나 잘린 부분이 있는가

### 3단계: `01_report.md` 작성

이미지에서 판독한 내용을 `01_report.md`에 상세히 기록했다.

이 보고서에는 다음 내용이 포함된다.

- 이미지별 화면 내용
- 이미지에 나온 입력 데이터
- 이미지에 나온 요구사항
- 출력 예시
- 화면 하단이 잘려서 확인할 수 없는 부분
- 판정 기준과 출력 예시가 서로 맞지 않는 부분

왜 이 작업을 하는가?

이미지를 바로 보고 코드를 작성하면 놓치는 부분이 생기기 쉽다. 먼저 보고서를 만들어 요구사항을 글로 정리하면 이후 작업이 명확해진다.

즉, `01_report.md`는 이미지 내용을 사람이 읽기 쉬운 문서로 바꾸는 단계다.

### 4단계: `02_task.md` 작성

`01_report.md`와 이미지 내용을 바탕으로 실제로 해야 할 일을 `02_task.md`에 정리했다.

`02_task.md`에는 다음 내용이 들어간다.

- 첫 번째 과제에서 해야 할 일
- 두 번째 과제에서 해야 할 일
- 계산 공식
- 정렬 기준
- 출력해야 하는 리포트 구조
- 구현 시 주의해야 할 점

왜 이 작업을 하는가?

보고서는 이미지 내용 전체를 상세히 담는 문서이고, 작업 문서는 실제 개발자가 구현해야 할 내용을 정리한 문서다.

예를 들어 보고서에는 이미지 전체 설명이 들어가지만, 작업 문서에는 다음처럼 실습자가 코딩할 때 필요한 핵심이 정리된다.

```text
예상 소요 일수 = 작업 소요 시간 / 8
남은 일수 = 마감일 - 기준일
남은 일수 < 예상 소요 일수 이면 위험
```

이런 식으로 해야 할 일을 정리해 두면 Python 코드를 만들 때 기준이 흔들리지 않는다.

### 5단계: `src` 폴더 생성

Python 실습 코드는 `src` 폴더 아래에 만들기로 했다.

`src`는 source의 줄임말로, 실제 실행되는 소스 코드 파일을 넣는 폴더로 자주 사용된다.

초보자가 직접 폴더를 만들려면 다음 명령어를 입력한다.

```bash
mkdir -p src
```

명령어 뜻은 다음과 같다.

- `mkdir`: 폴더를 만든다
- `-p`: 이미 폴더가 있어도 오류를 내지 않고 넘어간다
- `src`: 만들 폴더 이름

정상 생성 여부는 다음 명령어로 확인한다.

```bash
ls
```

또는 다음 명령어로 폴더만 확인할 수 있다.

```bash
find . -maxdepth 1 -type d
```

정상이라면 `./src`가 보여야 한다.

### 6단계: `src/practice.py` 생성

`02_task.md`에 정리된 내용을 바탕으로 실제 Python 실습 파일을 만들었다.

생성한 파일은 다음과 같다.

```text
src/practice.py
```

왜 이 파일을 만드는가?

`02_task.md`는 해야 할 일을 설명하는 문서이고, `practice.py`는 그 일을 실제로 수행하는 Python 코드다.

즉, 문서에 적힌 요구사항을 컴퓨터가 실행할 수 있는 코드로 바꾸는 단계다.

## 4. Python 코드 작성 방법

이 실습 코드는 크게 두 부분으로 나뉜다.

1. 작업 리스크 및 우선순위 분석
2. 통합 운영 대시보드 분석

각 부분을 함수로 나누어 작성했다.

함수란 특정 작업을 수행하는 코드 묶음이다. 예를 들어 날짜를 변환하는 작업, 위험 작업을 분석하는 작업, KPI를 계산하는 작업을 각각 함수로 나누면 코드가 읽기 쉬워진다.

### 4.1 날짜 처리를 위한 준비

이미지의 과제에는 날짜 계산이 많이 나온다.

예를 들어 다음 계산이 필요하다.

```text
남은 일수 = 마감일 - 기준일
```

Python에서 날짜 계산을 하려면 `datetime`을 사용한다.

코드에서는 다음처럼 불러온다.

```python
from datetime import datetime
```

그리고 문자열 날짜를 날짜 객체로 바꾸는 함수를 만든다.

```python
def parse_date(date_text):
    return datetime.strptime(date_text, "%Y-%m-%d")
```

왜 필요한가?

`"2026-01-30"` 같은 값은 그냥 글자다. 글자끼리는 날짜 차이를 계산할 수 없다. 그래서 Python이 날짜로 이해할 수 있게 변환해야 한다.

### 4.2 작업 우선순위 기준 만들기

첫 번째 과제에서는 우선순위가 다음 순서라고 했다.

```text
urgent > high > medium > low
```

Python에서 정렬하기 쉽게 숫자로 바꿔 준다.

```python
PRIORITY_ORDER = {
    "urgent": 0,
    "high": 1,
    "medium": 2,
    "low": 3,
}
```

왜 `urgent`가 0인가?

Python 정렬은 숫자가 작을수록 앞에 온다. 그래서 가장 먼저 처리해야 하는 `urgent`에 0을 준다.

### 4.3 작업 리스크 분석 코드 만들기

작업 리스크 분석은 다음 순서로 처리한다.

1. 각 작업의 마감일을 날짜로 변환한다.
2. 기준일과 마감일의 차이를 계산해 남은 일수를 구한다.
3. 작업 소요 시간을 8로 나눠 예상 소요 일수를 구한다.
4. 남은 일수가 예상 소요 일수보다 작으면 위험으로 판단한다.
5. 우선순위와 마감일 기준으로 작업을 정렬한다.
6. 전체 작업 수, 위험 작업 수, 정상 작업 수, 총 작업 시간을 계산한다.

핵심 공식은 다음과 같다.

```text
예상 소요 일수 = 작업 소요 시간 / 8
남은 일수 = 마감일 - 기준일
남은 일수 < 예상 소요 일수 이면 위험
```

이 기준으로 `analyze_task_risk()` 함수를 만든다.

함수의 역할은 다음과 같다.

- 입력: 작업 목록과 기준일
- 처리: 리스크 계산, 정렬, 요약 생성
- 출력: 분석된 작업 목록, 정렬된 작업 목록, 요약 정보

### 4.4 작업 리포트 출력 코드 만들기

분석만 하고 끝내면 사람이 결과를 보기 어렵다. 그래서 결과를 화면에 출력하는 함수가 필요하다.

`print_task_report()` 함수는 다음 내용을 출력한다.

```text
=== 작업 리스크 및 우선순위 분석 리포트 ===

[위험 작업]
[작업 우선순위]
[요약]
```

초보자는 여기서 `print()`를 이해하면 된다.

`print()`는 화면에 글자를 출력하는 Python 명령어다.

예를 들어 다음 코드는 화면에 `안녕하세요`를 출력한다.

```python
print("안녕하세요")
```

### 4.5 대시보드 데이터 만들기

두 번째 과제는 `dashboard_data`라는 데이터를 사용한다.

데이터는 다음 네 묶음으로 나뉜다.

- `dataset1`: 시간대별 전체 건수와 오류 건수
- `dataset2`: 항목별 현재 수량, 최소 필요량, 최대 허용량, 일일 사용량
- `dataset3`: 작업 일정 데이터
- `dataset4`: 시간별 측정값과 상태값

Python에서는 여러 데이터를 묶을 때 딕셔너리와 리스트를 사용한다.

예시는 다음과 같다.

```python
dashboard_data = {
    "dataset1": [
        {"hour": 0, "total": 120, "errors": 5},
        {"hour": 1, "total": 115, "errors": 3},
    ],
    "date": "2026-01-26",
}
```

여기서 알아야 할 것은 다음 정도다.

- `{}`는 딕셔너리다.
- `[]`는 리스트다.
- 딕셔너리는 이름표가 붙은 데이터 묶음이다.
- 리스트는 여러 개의 값을 순서대로 담는 묶음이다.

### 4.6 KPI 계산 코드 만들기

두 번째 과제에서 계산해야 하는 KPI는 다음과 같다.

- 오류율
- 항목 적정성
- 마감 가능성
- 정상 상태 비율
- 종합 점수

각 계산 방법은 다음과 같다.

#### 오류율

```text
오류율 = 전체 오류 수 / 전체 처리 수 * 100
```

예를 들어 전체 처리 수가 730이고 오류 수가 41이면 다음과 같다.

```text
41 / 730 * 100 = 5.6%
```

#### 항목 적정성

항목의 현재 수량이 최소 필요량 이상이고 최대 허용량 이하이면 적정하다고 판단한다.

```text
min_required <= current <= max_allowed
```

전체 항목 중 적정 항목의 비율을 계산한다.

#### 마감 가능성

작업마다 다음 기준으로 마감 가능 여부를 판단한다.

```text
남은 일수 >= 예상 소요 일수
```

전체 작업 중 마감 가능한 작업의 비율을 계산한다.

#### 정상 상태 비율

`dataset4`에서 `status`가 `normal`인 데이터의 비율을 계산한다.

```text
정상 상태 비율 = normal 개수 / 전체 상태 개수 * 100
```

#### 종합 점수

종합 점수는 오류율 점수, 항목 적정성, 마감 가능성, 정상 상태 비율의 평균으로 계산했다.

```text
종합 점수 = 4개 지표의 평균
```

### 4.7 긴급 이슈 TOP 5 만들기

대시보드 과제에서는 단순히 KPI만 출력하는 것이 아니라 긴급 대응 필요 이슈 TOP 5를 뽑아야 한다.

코드에서는 다음 유형을 이슈로 추출했다.

- 오류율 목표 초과
- 항목 부족
- 작업 일정 지연 위험
- 비정상 데이터 감지
- 종합 점수 목표 미달

각 이슈에는 점수를 부여하고, 점수가 높은 순서대로 정렬한다.

왜 점수를 부여하는가?

여러 문제가 동시에 생길 수 있기 때문이다. 모든 문제를 한 번에 처리하기 어렵다면 더 위험하고 영향이 큰 문제부터 처리해야 한다. 그래서 위험도와 영향도를 반영한 점수를 기준으로 순서를 정한다.

### 4.8 개선 권고사항 만들기

KPI가 목표치보다 낮으면 개선 권고사항을 만든다.

예를 들어 오류율이 목표보다 높으면 다음과 같은 권고사항을 출력한다.

```text
오류율 개선: 오류가 집중된 시간대의 처리 로직과 데이터 수집 경로를 점검한다.
예상 효과: 오류율 5% 이하 회복.
우선순위: 높음
```

왜 필요한가?

분석 결과만 보여 주면 사용자는 무엇을 해야 할지 모를 수 있다. 그래서 문제와 함께 해결 행동까지 제안해야 실무형 대시보드가 된다.

## 7. 구현 결과 해석

### 7.1 작업 리스크 분석 결과

공식 판정식 기준으로 실행하면 위험 작업은 `작업D` 1건이다.

이유는 다음과 같다.

```text
작업D
마감일: 2026-01-27
기준일: 2026-01-26
남은 일수: 1일
작업 시간: 10시간
예상 소요 일수: 10 / 8 = 1.25일
```

위험 판정 기준은 다음과 같다.

```text
남은 일수 < 예상 소요 일수
```

따라서 `1일 < 1.25일`이므로 `작업D`는 위험이다.

### 7.2 `작업C` 관련 주의점

이미지의 예시에는 `작업C`가 위험 작업으로 표시되어 있다.

하지만 공식 판정식대로 계산하면 `작업C`는 위험 작업이 아니다.

```text
작업C
마감일: 2026-01-28
기준일: 2026-01-26
남은 일수: 2일
작업 시간: 4시간
예상 소요 일수: 4 / 8 = 0.5일
```

판정식에 넣으면 다음과 같다.

```text
2일 < 0.5일
```

이 조건은 거짓이다. 따라서 코드에서는 `작업C`를 정상으로 처리했다.

왜 이렇게 처리했는가?

과제에서 명시한 공식 기준이 출력 예시보다 우선한다고 판단했기 때문이다. 출력 예시는 사람이 작성하면서 실수했을 가능성이 있지만, 공식은 구현 기준으로 직접 제시되어 있다.

### 7.3 대시보드 KPI 결과

대시보드 실습에서는 다음 KPI를 계산한다.

```text
오류율: 5.6%
항목 적정성: 50.0%
마감 가능성: 100.0%
정상 상태율: 50.0%
종합 점수: 73.5점
```

각 결과의 의미는 다음과 같다.

- 오류율 5.6%: 목표인 5.0%보다 높으므로 주의가 필요하다.
- 항목 적정성 50.0%: 전체 항목 중 절반만 기준을 충족하므로 위험이다.
- 마감 가능성 100.0%: 현재 데이터 기준으로 마감 가능한 작업 비율은 정상이다.
- 정상 상태율 50.0%: 상태 데이터 중 절반만 정상이라 위험이다.
- 종합 점수 73.5점: 목표 90점보다 낮으므로 개선이 필요하다.

## 8. 초보자가 코딩할 때 주의할 점

### 8.1 들여쓰기

Python은 들여쓰기가 매우 중요하다.

예를 들어 다음 코드는 정상이다.

```python
if True:
    print("정상")
```

하지만 다음 코드는 오류가 난다.

```python
if True:
print("오류")
```

`print()` 앞에 들여쓰기가 없기 때문이다.

### 8.2 따옴표

문자열은 따옴표로 감싸야 한다.

정상 예시:

```python
name = "작업A"
```

오류 예시:

```python
name = 작업A
```

### 8.3 콜론

`def`, `if`, `for` 같은 문장 끝에는 콜론(`:`)이 필요하다.

정상 예시:

```python
def main():
    print("실행")
```

오류 예시:

```python
def main()
    print("실행")
```

### 8.4 괄호

괄호는 열었으면 반드시 닫아야 한다.

정상 예시:

```python
print("hello")
```

오류 예시:

```python
print("hello"
```

### 8.5 파일 위치

실행 명령어는 프로젝트 최상위 폴더에서 입력해야 한다.

정상 위치:

```text
/home/manic/projects/python_new
```

정상 실행 명령:

```bash
python3 src/practice.py
```

만약 `src` 폴더 안으로 들어간 상태라면 실행 명령이 달라진다.

```bash
python3 practice.py
```

초보자는 혼동을 줄이기 위해 항상 프로젝트 최상위 폴더에서 실행하는 것이 좋다.
