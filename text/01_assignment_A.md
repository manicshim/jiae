# 과제 A 이미지 문자열 정리

대상 이미지:

- `img/s1.jpg`
- `img/s2.jpg`

## 1. 과제 제목

작업 리스크 및 우선순위 분석 프로그램

## 2. 과제 내용

여러 작업(Task) 데이터를 기반으로 업무 리스크를 분석하고 처리 우선순위를 도출하는 프로그램을 개발해야 한다.

각 작업은 예상 소요 시간, 마감일, 우선순위 정보를 가진다.

입력 데이터를 분석하여 위험 작업을 식별하고, 우선 처리 순서를 정리하며, 전체 작업 현황을 요약해야 한다.

## 3. 입력 데이터

```python
from datetime import datetime

tasks = [
    {
        "task_id": "T001",
        "name": "작업A",
        "workload_hours": 8,
        "due_date": "2026-01-30",
        "priority": "high"
    },
    {
        "task_id": "T002",
        "name": "작업B",
        "workload_hours": 12,
        "due_date": "2026-02-05",
        "priority": "medium"
    },
    {
        "task_id": "T003",
        "name": "작업C",
        "workload_hours": 4,
        "due_date": "2026-01-28",
        "priority": "urgent"
    },
    {
        "task_id": "T004",
        "name": "작업D",
        "workload_hours": 10,
        "due_date": "2026-01-27",
        "priority": "high"
    },
    {
        "task_id": "T005",
        "name": "작업E",
        "workload_hours": 6,
        "due_date": "2026-02-10",
        "priority": "low"
    }
]
```

## 4. 기준일

```python
base_date = datetime.strptime("2026-01-26", "%Y-%m-%d")
```

## 5. 작업 리스크 분석 기준

예상 소요 일수는 작업 소요 시간을 8로 나누어 계산한다.

```text
예상 소요 일수 = 작업 소요 시간 / 8
```

남은 일수는 마감일에서 기준일을 빼서 계산한다.

```text
남은 일수 = 마감일 - 기준일
```

남은 일수가 예상 소요 일수보다 작으면 위험 작업으로 판단한다.

```text
남은 일수 < 예상 소요 일수 -> 위험
```

그 외의 작업은 정상 작업으로 판단한다.

## 6. 우선순위 정렬 기준

작업 우선순위는 다음 순서로 정렬한다.

```text
urgent > high > medium > low
```

동일 우선순위 내에서는 마감일이 빠른 작업을 우선한다.

## 7. 작업 현황 요약 항목

전체 작업 수를 출력해야 한다.

위험 작업 수를 출력해야 한다.

정상 작업 수를 출력해야 한다.

총 예상 작업 시간을 시간 단위로 출력해야 한다.

## 8. 출력 예시

```text
=== 작업 리스크 및 우선순위 분석 리포트 ===

[위험 작업] 2건
작업: 작업C (T003)
남은 일수: 2일 / 소요 일수: 0.5일
작업: 작업D (T004)
남은 일수: 1일 / 소요 일수: 1.3일

[작업 우선순위]
1. 작업C (T003) - urgent, 마감 2026-01-28
2. 작업D (T004) - high, 마감 2026-01-27
3. 작업A (T001) - high, 마감 2026-01-30
4. 작업B (T002) - medium, 마감 2026-02-05
5. 작업E (T005) - low, 마감 2026-02-10

[요약]
전체 작업 수: 5건
위험 작업 수: 2건
정상 작업 수: 3건
총 작업 시간: 40시간
```

## 9. 문자열 판독 메모

이미지의 출력 예시에는 `작업C`가 위험 작업으로 포함되어 있다.

그러나 이미지에 적힌 공식 기준대로 계산하면 `작업C`는 위험 작업이 아니다.

`작업C`는 기준일 `2026-01-26` 기준으로 마감일까지 2일이 남아 있고, 작업 소요 시간은 4시간이므로 예상 소요 일수는 0.5일이다.

따라서 `2일 < 0.5일` 조건은 거짓이다.

공식 기준만 따르면 `작업D`가 위험 작업이다.
