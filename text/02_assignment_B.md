# 과제 B 이미지 문자열 정리

대상 이미지:

- `img/s3.jpg`
- `img/s4.jpg`
- `img/s5.jpg`

## 1. 과제 제목

통합 운영 대시보드 시스템

## 2. 과제 내용

조직은 여러 데이터 소스를 실시간으로 통합 분석하여 운영 상태를 파악해야 한다.

또한 데이터 기반 의사결정을 지원하는 대시보드 시스템이 필요하다.

다양한 지표를 계산하고, 잠재적 위험 요소를 식별하며, 개선을 위한 권고사항을 자동으로 생성해야 한다.

## 3. 입력 데이터 구조

입력 데이터는 `dashboard_data` 딕셔너리 형태로 제시되어 있다.

데이터는 `dataset1`, `dataset2`, `dataset3`, `dataset4`, `date`로 구성된다.

## 4. dataset1

`dataset1`은 시간대별 전체 건수와 오류 건수를 담고 있다.

```python
"dataset1": [
    {"hour": 0, "total": 120, "errors": 5},
    {"hour": 1, "total": 115, "errors": 3},
    {"hour": 2, "total": 118, "errors": 8},
    {"hour": 3, "total": 125, "errors": 15},
    {"hour": 4, "total": 122, "errors": 4},
    {"hour": 5, "total": 130, "errors": 6}
]
```

## 5. dataset2

`dataset2`는 항목별 현재값, 최소 필요값, 최대 허용값, 일일 사용량을 담고 있다.

```python
"dataset2": [
    {
        "id": "I001",
        "name": "항목1",
        "current": 50,
        "min_required": 100,
        "max_allowed": 500,
        "daily_usage": 15
    },
    {
        "id": "I002",
        "name": "항목2",
        "current": 5000,
        "min_required": 1000,
        "max_allowed": 8000,
        "daily_usage": 200
    }
]
```

## 6. dataset3

`dataset3`은 작업 일정 데이터를 담고 있다.

이미지에서 확인되는 작업 데이터는 다음과 같다.

```python
"dataset3": [
    {
        "task_id": "T001",
        "name": "작업A",
        "due_date": "2026-01-30",
        "priority": "high",
        "workload_hours": 8
    },
    {
        "task_id": "T003",
        "name": "작업C",
        "due_date": "2026-01-28",
        "priority": "urgent",
        "workload_hours": 4
    }
]
```

## 7. dataset4

`dataset4`는 시간별 측정값과 상태 정보를 담고 있다.

```python
"dataset4": [
    {
        "time": "03:00",
        "value1": 95,
        "value2": 4.2,
        "value3": 6.1,
        "status": "warning"
    },
    {
        "time": "04:00",
        "value1": 88,
        "value2": 3.5,
        "value3": 5.8,
        "status": "normal"
    }
],
"date": "2026-01-26"
```

## 8. 요구사항 1: 핵심 운영 지표(KPI) 계산

오류율을 계산해야 한다.

오류율은 `dataset1`을 기반으로 계산한다.

항목 적정성을 계산해야 한다.

항목 적정성은 `dataset2`에서 기준 충족 항목 비율로 계산한다.

마감 가능성을 계산해야 한다.

마감 가능성은 `dataset3`에서 마감일까지 완료 가능한 작업 비율로 계산한다.

정상 상태 비율을 계산해야 한다.

정상 상태 비율은 `dataset4`에서 `status='normal'`인 데이터 비율로 계산한다.

종합 점수를 계산해야 한다.

종합 점수는 4개 지표의 가중 평균으로 계산하며, 100점 만점 기준이다.

## 9. 요구사항 2: 긴급 대응 필요 이슈 우선순위 정렬

`dataset1`부터 `dataset4`까지 4개 이상의 이슈 유형을 추출해야 한다.

위험도와 영향도 기반 점수를 계산해야 한다.

상위 5개 이슈를 출력해야 한다.

## 10. 요구사항 3: AI 기반 개선 권고사항 생성

KPI가 목표치에 미달하면 구체적인 개선 액션을 제시해야 한다.

개선 권고사항에는 예상 효과와 우선순위를 포함해야 한다.

## 11. 목표치

오류율 목표치는 5% 이하이다.

```text
오류율 <= 5%
```

항목 적정성 목표치는 100%이다.

```text
항목 적정성 = 100%
```

마감 가능성 목표치는 95% 이상이다.

```text
마감 가능성 >= 95%
```

정상 상태 비율 목표치는 90% 이상이다.

```text
정상 상태 비율 >= 90%
```

## 12. 출력 예시

```text
# 과제 B: 통합 운영 대시보드 시스템

- 기준일: 2026-01-26

## 핵심 운영 지표 (KPI)

| 지표 | 현재값 | 목표값 | 상태 |
| --- | ---: | ---: | --- |
| 오류율 | 5.6% | 5.0% | 주의 |
| 항목 적정성 | 50.0% | 100% | 위험 |
| 마감 가능성 | 100.0% | 95.0% | 정상 |
| 정상 상태율 | 50.0% | 90.0% | 위험 |
| 종합 점수 | 73.5점 | 90점 | 주의 |

## 긴급 대응 필요 이슈 TOP 5

1. **[크리티컬] 작업 일정 지연 위험**
   - 내용: T003: 마감 2일 전, 소요 0.5일
   - 조치: 우선 처리 필요
2. **[크리티컬] 비정상 데이터 감지**
   - 내용: value1/value2/value3 급변 또는 warning 상태 감지
   - 조치: 즉시 점검
3. **[크리티컬] 오류율 목표 초과**
   - 내용: 현재 오류율 5.6%, 목표 5.0%
   - 조치: 오류 발생 시간대를 우선 점검
4. **[중요] 항목 부족**
   - 내용: 항목1: 기준 미달, 3.3일 후 소진 예상
   - 조치: 보충 필요
5. **[중요] 종합 점수 목표 미달**
   - 내용: 현재 종합 점수 73.5점, 목표 90점
   - 조치: 미달 KPI별 개선 계획 수립

## AI 기반 개선 권고사항

1. **오류율 개선**
   - 개선 액션: 오류가 집중된 시간대의 처리 로직과 데이터 수집 경로를 점검한다.
   - 예상 효과: 오류율 5% 이하 회복
   - 우선순위: 높음
2. **항목 적정성 개선**
   - 개선 액션: 기준 미달 항목을 즉시 보충하고 최소 보유량 알림을 설정한다.
   - 예상 효과: 항목 적정성 100% 달성
   - 우선순위: 높음
3. **정상 상태율 개선**
   - 개선 액션: warning 상태의 측정값을 즉시 점검하고 임계치 초과 원인을 분석한다.
   - 예상 효과: 정상 상태율 90% 이상 회복
   - 우선순위: 중간
```

## 13. 문자열 판독 메모

`img/s3.jpg`는 화면 하단 일부가 버튼 영역에 가려져 있다.

`img/s5.jpg`의 출력 예시는 세 번째 이슈까지만 화면에 보인다.

이미지에는 `[긴급 대응 필요 이슈] TOP 5`라고 되어 있으므로 네 번째와 다섯 번째 이슈 또는 후속 권고사항이 아래에 있을 가능성이 있다.

이 문자열 정리는 이미지에서 판독 가능한 내용을 기준으로 작성했다.

현재 Python 구현은 공식 입력값 기준으로 KPI와 이슈를 다시 계산하며, Markdown 표와 목록 형식으로 결과를 출력한다.

## 14. 현재 구현 기준

현재 구현은 다음 기준으로 정리되어 있다.

- 함수 상단 주석은 입력, 계산, 출력 역할이 드러나도록 작성한다.
- 출력 결과는 Markdown 표와 목록 형식으로 통일한다.
- KPI, 이슈, 권고사항은 공통 계산 결과를 재사용해 일관성을 유지한다.
