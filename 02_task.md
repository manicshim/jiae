# 이미지 과제 내용 설명

`./img` 폴더의 이미지들은 크게 두 개의 코딩 과제 설명을 담고 있다.

## 1. 작업 리스크 및 우선순위 분석 프로그램

대상 이미지: `img/s1.jpg`, `img/s2.jpg`

이 과제는 `tasks`라는 작업 목록을 가지고 업무 리스크를 분석하고, 처리 우선순위를 정리하는 프로그램을 만들라는 내용이다.

해야 할 일은 다음과 같다.

- 각 작업의 남은 일수 계산
- 각 작업의 작업 소요 일수 계산
- 마감일까지 끝낼 수 있는지 위험 여부 판단
- 우선순위 기준으로 작업 정렬
- 전체 작업 현황 요약 출력

핵심 계산 기준은 다음과 같다.

```text
예상 소요 일수 = 작업 소요 시간 / 8
남은 일수 = 마감일 - 기준일
남은 일수 < 예상 소요 일수 이면 위험
그 외는 정상
```

작업 우선순위는 다음 순서로 정렬한다.

```text
urgent > high > medium > low
```

같은 우선순위 안에서는 마감일이 빠른 작업을 먼저 처리한다.

최종적으로 출력해야 하는 보고서 구성은 다음과 같다.

```text
[위험 작업]
[작업 우선순위]
[요약]
```

주의할 점: 이미지의 출력 예시에는 `작업C`가 위험 작업으로 들어가 있지만, 제시된 판정식대로라면 `작업C`는 위험 작업이 아니다. 기준일이 `2026-01-26`, 마감일이 `2026-01-28`, 소요 시간이 4시간이므로 남은 일수는 2일이고 예상 소요 일수는 0.5일이다. 따라서 공식 기준만 보면 정상 작업이다. 실제 구현할 때는 과제에 적힌 공식 기준을 따르는 것이 타당하다.

## 2. 통합 운영 대시보드 시스템

대상 이미지: `img/s3.jpg`, `img/s4.jpg`, `img/s5.jpg`

이 과제는 `dashboard_data`라는 여러 데이터 묶음을 분석해서 운영 대시보드 리포트를 만드는 프로그램을 만들라는 내용이다.

데이터는 대략 다음처럼 나뉜다.

- `dataset1`: 시간대별 전체 건수와 오류 건수
- `dataset2`: 항목별 현재 수량, 최소 필요량, 최대 허용량, 일일 사용량
- `dataset3`: 작업 일정 데이터
- `dataset4`: 시간별 측정값과 정상/경고 상태

계산해야 하는 핵심 운영 지표(KPI)는 다음과 같다.

- 오류율
- 항목 적정성
- 마감 가능성
- 정상 상태 비율
- 종합 점수

추가로 해야 할 일은 다음과 같다.

- 긴급 대응 필요 이슈 TOP 5 추출
- KPI가 목표치보다 낮으면 개선 권고사항 생성
- 개선 권고사항에 예상 효과와 우선순위 포함

출력해야 하는 리포트 구성은 다음과 같다.

```text
[핵심 운영 지표 KPI]
[긴급 대응 필요 이슈 TOP 5]
[AI 기반 개선 권고사항]
```

목표치는 이미지에서 다음과 같이 확인된다.

```text
오류율 <= 5%
항목 적정성 = 100%
마감 가능성 >= 95%
정상 상태 비율 >= 90%
```

즉, 이 과제는 단순히 데이터를 출력하는 것이 아니라 여러 데이터셋을 통합해서 KPI를 계산하고, 위험 이슈를 찾아 우선순위를 매기고, 개선 액션까지 제안하는 대시보드형 분석 프로그램을 작성하라는 과제다.
