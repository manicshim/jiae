## 파일별 상세 내용

### `img/s1.jpg`

#### 화면 성격

AI 강의 과제 페이지의 `과제내용` 화면이다. 업무 리스크 분석 및 처리 우선순위 도출 프로그램 개발 과제가 제시되어 있다.

#### 과제 설명

여러 작업(Task) 데이터를 기반으로 업무 리스크를 분석하고 처리 우선순위를 도출하는 프로그램을 개발해야 한다. 각 작업에는 예상 소요 시간, 마감일, 우선순위가 포함된다.

#### 입력 데이터

Python 코드 형태의 작업 리스트가 보인다.

```python
from datetime import datetime

tasks = [
    {
        "task_id": "T001",
        "name": "작업A",
        "workload_hours": 8,
        "due_date": "2026-01-30",
        "priority": "high"
    },
    {
        "task_id": "T002",
        "name": "작업B",
        "workload_hours": 12,
        "due_date": "2026-02-05",
        "priority": "medium"
    },
    {
        "task_id": "T003",
        "name": "작업C",
        "workload_hours": 4,
        "due_date": "2026-01-28",
        "priority": "urgent"
    },
    {
        "task_id": "T004",
        "name": "작업D",
        "workload_hours": 10,
        "due_date": "2026-01-27",
        "priority": "high"
    },
    {
        "task_id": "T005",
        "name": "작업E",
        "workload_hours": 6,
        "due_date": "2026-02-10",
        "priority": "low"
    }
]
```

#### 기준일

기준일은 다음과 같이 제시되어 있다.

```python
base_date = datetime.strptime("2026-01-26", "%Y-%m-%d")
```

#### 요구 결과

위 데이터를 분석하여 위험 작업을 식별하고, 우선 처리 순서를 정리하며, 전체 작업 현황을 요약해야 한다.

### `img/s2.jpg`

#### 화면 성격

`s1.jpg`의 과제 설명이 이어지는 화면이다. 작업 리스크 판정 기준, 우선순위 정렬 기준, 현황 요약 항목, 출력 예시가 포함되어 있다.

#### 작업 리스크 분석 기준

- 예상 소요 일수 = 작업 소요 시간 / 8
- 남은 일수 = 마감일 - 기준일
- 남은 일수 < 예상 소요 일수이면 위험
- 그 외는 정상

#### 우선순위 기반 작업 정렬 기준

우선순위 순서는 다음과 같다.

```text
urgent > high > medium > low
```

동일 우선순위 내에서는 마감일이 빠른 작업을 우선한다.

#### 작업 현황 요약 항목

다음 항목을 요약해야 한다.

- 전체 작업 수
- 위험 작업 수
- 정상 작업 수
- 총 예상 작업 시간(시간 단위)

#### 출력 예시

화면에 보이는 출력 예시는 다음과 같다.

```text
=== 작업 리스크 및 우선순위 분석 리포트 ===

[위험 작업] 2건
작업: 작업C (T003)
남은 일수: 2일 / 소요 일수: 0.5일
작업: 작업D (T004)
남은 일수: 1일 / 소요 일수: 1.3일

[작업 우선순위]
1. 작업C (T003) - urgent, 마감 2026-01-28
2. 작업D (T004) - high, 마감 2026-01-27
3. 작업A (T001) - high, 마감 2026-01-30
4. 작업B (T002) - medium, 마감 2026-02-05
5. 작업E (T005) - low, 마감 2026-02-10

[요약]
전체 작업 수: 5건
위험 작업 수: 2건
정상 작업 수: 3건
총 작업 시간: 40시간
```

#### 관찰 메모

예시에서 `작업C`는 남은 일수 2일, 소요 일수 0.5일로 표시되어 있으므로 리스크 판정 기준상 정상이어야 한다. 그러나 출력 예시에는 위험 작업으로 포함되어 있다. 반면 `작업D`는 남은 일수 1일, 소요 일수 1.25일이므로 위험 작업으로 판단된다. 이미지의 예시는 과제 설명용 샘플일 수 있으나, 제시된 판정식과 출력 예시 사이에 불일치가 있다.

### `img/s3.jpg`

#### 화면 성격

새로운 `과제내용` 화면으로 보인다. 조직의 여러 데이터 소스를 실시간 통합 분석하여 운영 상태를 파악하고, 데이터 기반 의사결정을 지원하는 대시보드 시스템 개발 과제다.

#### 과제 설명

조직은 여러 데이터 소스를 실시간으로 통합 분석하여 운영 상태를 파악하고, 데이터 기반 의사결정을 지원하는 대시보드 시스템이 필요하다. 다양한 지표를 계산하고, 잠재적 위험 요소를 식별하며, 개선을 위한 권고사항을 자동으로 생성해야 한다.

#### 입력 데이터 구조

`dashboard_data` 딕셔너리 형태의 데이터가 제시되어 있다.

##### `dataset2`


### `img/s4.jpg`

#### 화면 성격

시간별 측정값과 상태가 포함된 데이터가 보인다.

```python
"dataset4": [
    {
        "time": "03:00",
```

#### 요구사항

##### 1. 핵심 운영 지표(KPI) 계산

아래 항목을 계산하는 프로그램을 작성해야 한다.

- 오류율: `dataset1` 기반
- 항목 적정성: `dataset2`에서 기준 충족 항목 비율
- 마감 가능성: `dataset3`에서 마감일까지 완료 가능한 작업 비율
- 정상 상태 비율: `dataset4`에서 `status='normal'` 비율
- 종합 점수: 4개 지표의 가중 평균, 100점 만점

##### 2. 긴급 대응 필요 이슈 우선순위 정렬

- `dataset1`부터 `dataset4`까지 4개 이상의 이슈 유형 추출
- 위험도 + 영향도 기반 점수 계산
- 상위 5개 이슈 출력

##### 3. AI 기반 개선 권고사항 생성

- KPI가 목표치 미달 시 구체적 개선 액션 제시
- 예상 효과 및 우선순위 포함

#### 목표치

화면에 보이는 목표치는 다음과 같다.

- 오류율 <= 5%
- 항목 적정성 = 100%

나머지 목표치는 `s5.jpg`에 이어진다.

### `img/s5.jpg`

## 전체 정리

이미지들은 크게 두 개의 과제 내용을 담고 있다.

1. 작업 리스크 및 우선순위 분석 프로그램
   - 입력: `tasks` 리스트와 기준일 `2026-01-26`
   - 핵심 로직: 소요 일수와 남은 일수를 비교해 위험 작업 판정
   - 정렬 기준: `urgent > high > medium > low`, 동일 우선순위는 빠른 마감일 우선
   - 출력: 위험 작업, 작업 우선순위, 전체 요약

2. 통합 운영 대시보드 시스템
   - 입력: `dashboard_data`의 `dataset1`~`dataset4`
   - 핵심 로직: 오류율, 항목 적정성, 마감 가능성, 정상 상태율, 종합 점수 계산
   - 추가 요구: 긴급 이슈 TOP 5 추출, KPI 미달 시 개선 권고사항 생성
   - 출력: KPI 표, 긴급 대응 필요 이슈 목록, 개선 액션

## 주의할 점

- `s2.jpg`의 위험 작업 출력 예시는 제시된 판정식과 일부 불일치한다. 판정식대로라면 `작업C`는 남은 일수 2일, 소요 일수 0.5일이므로 위험이 아니다.
- `s3.jpg`와 `s5.jpg`는 화면 하단 일부가 잘려 있어 전체 데이터와 전체 출력 예시가 완전히 보이지 않는다.
- 모든 내용은 이미지에서 판독 가능한 텍스트를 기준으로 작성했으며, 이미지 밖에 있는 숨겨진 스크롤 영역은 확인할 수 없다.
